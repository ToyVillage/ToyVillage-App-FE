# Scenario Draft — close-schedule-edit

출처: `harness/specs/close-schedule-edit.spec.md`

## 핵심 시나리오

### S1: 목록 카드에서 수정 진입

- Given: 오른쪽 휴관 일정 목록이 보인다.
- When: 일정 카드를 클릭한다.
- Then: 해당 ID의 수정 화면으로 이동한다.

### S2: 기존 값 표시

- Given: 수정할 휴관 일정이 존재한다.
- When: 수정 화면에 진입한다.
- Then: 기존 시작일, 종료일과 제목이 입력되어 있다.

### S3: 수정 성공

- Given: 유효한 날짜와 제목을 입력했다.
- When: `수정하기`를 클릭한다.
- Then: 목록으로 이동하고 동일 ID 카드 하나에 수정값이 보인다.

### S4: 수정 검증

- Given: 날짜가 비었거나 순서가 잘못됐거나 제목이 비어 있다.
- When: `수정하기`를 클릭한다.
- Then: 수정하지 않고 첫 오류 dialog를 표시한다.

## 엣지 케이스

### S5: 존재하지 않는 ID

- Given: 존재하지 않는 일정 ID의 수정 URL이다.
- When: 페이지가 일정을 조회한다.
- Then: 휴관일 관리로 replace 이동한다.

### S6: 수정 실패 시 입력 보존

- Given: 유효한 값이 입력되어 있고 저장소가 실패한다.
- When: `수정하기`를 클릭한다.
- Then: URL과 입력값을 유지하고 다시 시도할 수 있다.

### S7: 키보드 접근

- Given: 휴관 일정 목록이 보인다.
- When: 키보드만으로 카드 진입, 제목 편집과 저장을 수행한다.
- Then: 수정된 값이 목록에 반영된다.
